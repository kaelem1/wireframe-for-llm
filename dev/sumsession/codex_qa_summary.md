## 环境配置
| 问题 | 解法 |
|---|---|
| 模型接入配置错误 | 把模型 ID 改成真实可用的配置，并从当前环境注入有效 API key，避免依赖不存在的模型或空密钥。 |
| 本地运行环境未就绪 | 先补齐 embedding 模型、API key 和基础服务，再处理 gateway 异常与系统限制，保证本地运行条件完整。 |
| gateway 进程冲突 | 启动前先清理旧 gateway 进程或切换端口，避免 `already running` 直接阻塞服务。 |
| 测试环境和权限受限 | 优先使用环境自带工具链和标准库测试；涉及沙箱边界或目录权限时，先修正路径或申请放开执行权限。 |
| 数据抓取入口不稳定 | 优先切到更稳定的数据入口，网关断开时先恢复代理或 gateway，再重试抓取。 |
| OpenClaw 网关与 Telegram 代理配置丢失 | 不要直接重写现有网关配置文件，保留代理字段并用确定的重启方式恢复服务。 |
| Git 远程仓库未配置 | 先为仓库配置 `remote` 和 `push destination`，再执行推送。 |
| pnpm 命令不可用 | 先恢复 `pnpm` 或 `corepack`，不行再切到项目已支持的包管理器。 |
| launchctl 环境变量不继承 | 将关键环境变量落到 `.env` 或服务实际读取的位置，不要依赖 `launchctl` 会话继承。 |
| Swift / Xcode / CLT 工具链失配 | 安装或切换到完整 Xcode，并统一 Command Line Tools、SDK 和 Swift 版本。 |
| 测试环境缺少 jsdom/HTMLElement | 前端测试切到 `jsdom` 环境后再运行，避免 DOM 全局对象缺失。 |

## 语法错误
| 问题 | 解法 |
|---|---|
| 脚本缺少常量定义 | 把运行路径依赖的常量补齐到源码里，避免真实调用时才因为未定义标识符崩溃。 |
| 命令参数拼接截断 | 重新按正确的参数边界拼接并执行命令，避免 shell 或脚本层截断。 |
| 类型签名与导出声明不一致 | 统一函数参数、类型定义和导出接口，先消除类型系统内部不一致，再继续构建或测试。 |

## 依赖冲突
| 问题 | 解法 |
|---|---|
| yfinance 依赖环境异常 | 先修复底层数值库或 Python 依赖环境，再继续跑 `yfinance` 相关逻辑。 |
| 导出接口与测试依赖不对齐 | 先补齐缺失导出，再把测试依赖按项目实际运行方式安装完整。 |

## API 调用
| 问题 | 解法 |
|---|---|
| 输出额度过高 | 收紧 `max_tokens` 和分段输出预算，避免远端按默认上限计费或触发额度压力。 |
| 请求体格式不兼容 | 按目标接口的兼容格式统一消息结构，避免服务端因为 schema 不匹配直接返回 500 或格式错误。 |
| LLM 回包格式不稳定 | 为 LLM 调用做统一封装，优先读取固定文本字段，再回退检查原始 JSON 结构。 |
| 代理链路调用不稳定 | 显式传入正确 agent 参数，保留原始响应，并在代理失败时按可控顺序回退。 |
| 插件长任务进度上报链路不清晰 | 通过 session store 与 `deliveryContext` 回传进度，不要只回工具名或空状态。 |
| institutional rewrite 调用失败 | 先修正请求参数类型和字段，再为失败场景保留确定性的本地 fallback。 |
| OpenClaw 调用与默认 agent 失败 | 先确认网关可用，再把默认 agent 改成真实存在的配置，并保留原始输出便于排查。 |
| Google API 参数无效 | 在发请求前先做 schema 清洗和参数校验，避免把无效字段直接发到上游。 |
| 批处理链路被限流和解析失败拖慢 | 收敛到低并发或单 worker，降低请求速率，并把模型输出解析稳定住后再扩大吞吐。 |

## 权限与认证
| 问题 | 解法 |
|---|---|
| 外部服务鉴权配置错误 | 统一修正 token、SecretRef 和凭据引用方式，替换失效令牌后再重启服务。 |
| 真实网络请求被权限边界拦截 | 需要真实外网时先放开对应执行权限，否则上游请求会在运行环境里直接失败。 |
| SILICONFLOW_API_KEY 缺失 | 先导出有效的 SiliconFlow key，再重新执行依赖该服务的流程。 |
| Gemini / ikungemini 授权失败 | 依次检查 OAuth 刷新状态、API key 和 User-Agent，先恢复授权链路再重试。 |

## 路径与文件
| 问题 | 解法 |
|---|---|
| 文件路径匹配失败 | 把路径匹配逻辑改成兼容真实文件名模式，不要假设固定路径或固定前缀。 |
| 技能路径符号链接越界 | 先做 `realpath` 解析，再按解析后的绝对路径校验是否仍在允许根目录内。 |
| 评估脚本与测试集缺失 | 先补齐脚本和测试集文件，再运行评估链路。 |
| 工作区核心文件缺失 | 把入口视图和核心子模块补全到约定位置，再恢复工作区主流程。 |
| 模块引用和源码路径不一致 | 统一实际目录结构与 import 路径，避免入口或补丁继续引用不存在的模块。 |

## 数据处理
| 问题 | 解法 |
|---|---|
| 检索逻辑过于单一 | 先做结构化意图分析，再拆成多组查询和结果归档，避免只用原话做单查询。 |
| 港股和 A 股数据 fallback 失败 | 为不同市场补上真实可用的数据源，并保留原始代码用于回退重试。 |
| HK 符号解析失败 | 先校验 HK 搜索返回负载的结构，再做遍历或字段提取。 |
| HK 财务回退字段不全 | 去掉只适用于美股的硬过滤，并补齐 HK 财务报表字段映射。 |
| market_cap 生成为空 | 单独修复 `market_cap` 生成逻辑，不要让空值继续参与后续计算。 |
| 财务口径混用导致历史估值错误 | 统一成单一财务口径后再计算估值，删除伪造的历史回推结果。 |
| 模型返回 JSON 解析失败 | 先修复本地解析逻辑并校验返回值完整性，保证进入后续流程前就是有效 JSON。 |
| 草稿未自动保存 | 把草稿从纯内存状态改成落盘持久化，避免重启或切页后内容丢失。 |
| Markdown 渲染失败无提示 | 对渲染失败给出显式错误或可见降级结果，不要静默回退。 |
| Reduce 汇总丢失大量结果 | 基于已有 map 原始结果重新汇总，避免 reduce 阶段把已提取内容大量丢弃。 |

## 逻辑错误
| 问题 | 解法 |
|---|---|
| 数据集验证逻辑不完整 | 补齐 opening prompt、sources 和 stop 条件校验，再用可重复的测试轮次验证流程收敛。 |
| 任务状态提取不足 | 在工具调用前提取状态文件路径，并优先消费工具返回值生成进度状态。 |
| 独立来源计数口径错误 | 按 `publisher`、`domain` 和可信度统一计算独立来源，不要把同站点重复算多次。 |
| 分支初始化与快照同步问题 | 分支已存在时也同步快照，并排除缓存产物，避免 overlay 丢失未提交内容。 |
| NSTextView 宽度约束为 0 | 把文本视图宽度与容器同步，并修正文档视图布局约束。 |
| 历史记录加载策略不合理 | 默认加载最近有数据的日期，并提供显式“加载更多”入口，不要依赖滚动高度碰运气。 |
| 编辑区与预览区分离 | 改成单区实时可视化输入，减少编辑和预览状态分叉。 |
| 日期标题与空记录展示不同步 | 统一日期格式，在跨天时主动刷新状态，并隐藏没有记录的日期标题。 |
| 画布菜单和拖动交互异常 | 收口弹层层级、指针事件和提交时机，保证菜单与拖拽状态一致。 |
| 组件归一化限制过严 | 只保留最小尺寸和画板边界等必要约束，放宽不必要的强制归一化。 |
| 文本编辑和复制交互不完整 | 用可持续编辑的输入控件替换一次性 prompt，并补齐画板级复制能力。 |

## 构建与运行
| 问题 | 解法 |
|---|---|
| 报告发布链路未打通 | 先把 Markdown 渲染成 HTML，再通过链接分发，避免直接发送超长文本。 |
| 脚本重写过程出错 | 使用更确定的整文件写入流程，避免补丁失败或转义破坏脚本。 |
| Package.swift / SwiftPM 清单链接失败 | 修正 `Package.swift`、`swift-tools-version` 和资源声明，保证 manifest 可被正确加载。 |
| SwiftPM 源码目录结构不规范 | 把源码放到 `Sources/目标名/` 的约定结构下，再让 SwiftPM 解析。 |
| 构建被类型错误和打包体积告警阻塞 | 先清掉类型错误，再做 Vite 拆包或资源压缩，避免一次把编译失败和体积告警混在一起处理。 |

## 其他
| 问题 | 解法 |
|---|---|
| 项目文档缺失 | 按仓库规则补齐 `.ai/current.md`、`.ai/decisions.md` 等必要文档。 |
| apiKey 配置字段类型错误 | 把 `apiKey` 调整为配置约定的字段类型和格式，避免配置校验直接失败。 |
| 编辑器深色模式可见性差 | 改用语义色并显式设置前景色，避免深色主题下出现白字白底。 |
| 保存/加载错误缺少用户提示 | 把失败信息提升为用户可见提示，而不是只写控制台日志。 |
| 右栏文案和样式不统一 | 统一交互文案、按钮布局和残留样式，避免同一侧栏出现多套规则。 |
